# P2-2 Greedy Algorithm Notes

## Goal

`P2-2` estimates how a 50-card deck behaves over the first several turns.

It does three things:

1. Enumerate possible hand states.
2. For each turn, choose plays greedily by maximizing current-turn `board_value`.
3. Output expected value and the most common play lines.

## Inputs

The current text-source pipeline reads:

- `data/cards/cards.yaml`
- `data/value_tables/value_table.yaml`
- `data/decks/*.txt`
- `data/scenarios/*.yaml`

## Turn Resources

The current first-player rule is:

- Turn `t`
- `LV = t + 1`
- `COST = t`

Examples:

- Turn 1: `LV=2`, `COST=1`
- Turn 2: `LV=3`, `COST=2`
- Turn 3: `LV=4`, `COST=3`

## Draw Model

- Opening hand: 6 cards
- Start of each turn: draw 1 card
- Deck size: 50 cards

The script tracks:

- current hand state
- remaining deck state
- probability of that state

## Play Legality

A card can be played only when all conditions are true:

- the hand contains at least one copy
- current `LV >= card LV`
- remaining `COST >= card COST`

## Greedy Rule

At each step, choose the legal play with the best current-turn score.

Current tie-break order:

1. `board_value`
2. `value`
3. `threat_power + threat_value`
4. `survival_power + survival_value`
5. lower `cost`

## Conditions

### Unconditional effects

These are counted directly.

### Resonance

Machine resonance is treated as a conditional effect.

Current simplified rule:

- if a matching pilot has already been played this turn, resonance is active
- if a matching pilot is still in hand and can still be played this turn, resonance is active
- otherwise, resonance is inactive

### Other conditions

If a condition is not modeled explicitly yet, the current fallback is:

- `50%`

## Current Scope

This script is a text-source starter, not a full Excel-parity engine yet.

Currently included:

- machine default value
- body value
- real machine resonance
- decklist input
- turn-state enumeration
- greedy play selection

Not fully migrated yet:

- advanced conditional effects
- graveyard state
- rest / active board state
- pilot attachment state
- shield / base state
- full burst logic
